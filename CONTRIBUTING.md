# Contributing to Algorithmic Quest

Thank you for your interest in contributing to this repository! This project serves as an automated workspace to track a progressive journey through Data Structures and Algorithms (DSA). 

We welcome contributions that improve algorithm efficiencies, add alternative language solutions, or fix structural bugs.

---

## 🗺️ How Can I Contribute?

### 1. Optimizing Existing Solutions
If you notice an approach with poor runtime metrics (like the 70 ms sync runtime) or heavy memory footprints, you can submit an optimized version.
* Keep the code clean and commented.
* Provide a brief explanation of why your approach optimizes the previous one.

### 2. Adding Alternative Languages
The repository primarily uses C++ and C. We highly welcome solutions written in:
* Python 🐍
* JavaScript / TypeScript 💻
* Java / Go / Rust ⚙️

### 3. Reporting Pipeline Issues
Since this repository relies heavily on automation scripts (`joshcai/leetcode-sync` and NeetCode native webhooks), please submit a traditional GitHub Issue if you spot synchronization failures or broken folder path routing.

---

## 🚀 The Pull Request Process

1. **Fork the Repository**: Create your own copy of this project to work on.
2. **Create a Branch**: Use a descriptive branch name (e.g., `feature/optimize-intersection-linked-list`).
3. **Commit Cleanly**: Write clear commit messages detailing your changes.
4. **Open a PR**: Fill out the Pull Request template completely. Ensure you state your solution's **Time Complexity** ($O(N)$) and **Space Complexity** ($O(1)$).

---

## 💬 Discussions vs. Issues

To keep the repository highly organized, please respect our communication channels:
* **Use Discussions:** For generalized questions, explaining alternative logic, general chat, or comparing platform runtimes.
* **Use Issues:** Strictly reserved for system bugs, error logs, or automation breaks.

Thank you for helping optimize this algorithmic journey! ⚔️
